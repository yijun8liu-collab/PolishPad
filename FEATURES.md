# PolishPad 功能清单（回归测试基准）

> 基准版本：提交 `85edc4f`（v0.5 稳定基线 + API Key 改存配置文件）。
> **用法**：新增/修改功能前，先圈出可能受影响的模块；改完后对照逐条验证。
> 同一功能的不同入口（快捷键 / 菜单 / 按钮 / 右键服务）**各算一条**，分别测试。
> 自动化冒烟：`swift tools/verify_baseline.swift`（核心链路 F1→F22→F24，
> 需要机器空闲、不要同时操作键鼠）；单元自检：`PolishPad --selftest`（25 项）；
> 提示词回归：`python3 tools/prompt_test.py`（改提示词后必跑，检查输出是改写而非回答）。

## 一、全局入口

| 编号 | 功能 | 触发方式 | 预期效果 |
|---|---|---|---|
| F1 | 唤起主面板 | 全局热键（默认 `ctrl+option+p`，可配置） | 面板出现在鼠标所在屏幕中上（类 Spotlight 位置），深色玻璃外观，**尺寸可调**（默认 680×400；组稿/审阅切换不变高）；**全新会话**；草稿框自动聚焦；记录唤起前的前台应用为粘贴目标 |
| F2 | 关闭主面板 | 面板开着时再按同一热键 | 面板关闭，焦点还给唤起前的应用；**进行中的请求同时取消**（与 Esc/红点一致） |
| F3 | 菜单栏菜单 | 点击菜单栏图标 | 弹出完整菜单（各项见 F4-F15）；**每次展开重建：文案跟随面板 中/EN 开关即时切换**，历史子菜单保持最新 |
| F4 | 打开优化窗口 | 菜单项（标题带当前热键） | 同 F1/F2 切换；注意：点菜单不会切走目标应用焦点 |
| F5 | 划词优化 | 全局热键（默认 `ctrl+option+r`），先选中文本 | HUD「优化中…」（流式字数进度）→ 优化结果**原地替换选中文本** → 恢复用户原剪贴板（含图片）→ Glass 音效 + HUD「已替换」；等修饰键物理松开才模拟 ⌘C（防叠加）；**结果返回时目标应用已不在前台 → 不盲贴，结果留剪贴板并提示手动粘贴**；**原文+结果写入历史（可从菜单找回被替换的原文）** |
| F6 | 划词优化 · 菜单入口 | 菜单「优化选中文本」 | 同 F5 |
| F7 | 划词优化 · 右键服务 | 目标应用中右键 → 服务 → 「PolishPad：优化并替换」 | 同 F5 |
| F8 | 全选优化 | 全局热键（默认 `ctrl+option+a`），光标在输入框内即可 | 先模拟 ⌘A 全选，其余同 F5；输入框为空时报错且不破坏原文 |
| F9 | 全选优化 · 菜单入口 | 菜单「全选优化替换」 | 同 F8 |
| F10 | 全选优化 · 右键服务 | 右键 → 服务 → 「PolishPad：全选优化并替换」 | 同 F8 |
| F11 | 还原上次替换 | 菜单「还原上次替换」 | 撤销最近一次 F5/F8 替换，恢复原文；**仅当目标应用仍在前台时执行**（防止按键误发到别的应用） |
| F12 | 历史 | 菜单「历史」子菜单 | 最近 20 条记录（**含面板会话和划词/全选优化**）；每条展开原文+各版本；点击任意版本复制到剪贴板 |
| F13 | 设置入口 · 菜单 | 菜单「设置…」 | 打开设置窗口（见模块五） |
| F14 | 打开配置文件 | 菜单「打开配置文件」 | 用默认编辑器打开 config.json |
| F15 | 退出 | 菜单「退出 PolishPad」（⌘Q） | 应用退出 |

## 二、主面板 · 组稿阶段

| 编号 | 功能 | 触发方式 | 预期效果 |
|---|---|---|---|
| F16 | 草稿输入 | 面板草稿框打字 | 多行输入；**Enter 提交、Shift+Enter 换行**；中文输入法组字中的回车交给输入法，不误提交 |
| F17 | Esc 分层退出 | Esc 键 | 录音中→仅停止录音；请求中→仅取消请求（恢复本轮前内容）；其他→关闭面板 |
| F18 | 语音听写 | 点击麦克风按钮 | 开始/停止录音；识别文字**追加**在框内已有文字之后（长停顿不丢前文）；识别语言固定跟设置 speechLocale（**不随 中/EN 开关变**——说中文出英文是核心场景） |
| F19 | 中/EN 切换 | 面板右上角 中/EN | 切换整个界面语言 + 输出语言（EN 用原生英文提示词，非中文提示词打补丁）；**菜单栏菜单、HUD、设置窗口（下次打开时）一并跟随**；选择被记住（重启保留） |
| F20 | 关闭按钮 | 面板**左上角红色圆点**（mac 风格，悬停显 ×） | 无条件收尾并关窗（停录音、取消请求） |
| F21 | 场景预设 | 底栏预设按钮（润色/Slack英文/正式/精简/自定义） | 唤起时按前台应用自动选中（appPresets 映射，显示"已按 X 自动选择"）；手动点击可切换；审阅阶段切换会用新场景重新生成当前文本 |
| F22 | 提交优化 | 草稿框 Enter | 流式显示结果 → 结果自动复制到剪贴板 → **自动粘贴回唤起时的前台应用**（HUD「已粘贴」）→ 焦点回到面板反馈框；每轮写入历史 |

## 三、主面板 · 审阅阶段（多轮纠偏）

|---|---|---|---|

| F27a | 快捷 chip「更短」 | 点击 chip | 按**修改**语义发送对应固定意见，其余同 F24 |
| F27b | 快捷 chip「更正式」 | 点击 chip | 同 F27a |

| F29 | 版本切换 | ⌘[ 上一版 / ⌘] 下一版 | 结果区显示对应版本并复制到剪贴板；回退后发反馈以**当前显示的版本**为基准 |

## 四、粘贴与替换机制（隐式行为）

| 编号 | 行为 | 说明 |
|---|---|---|
| F39 | 粘贴目标锁定 | 目标 = **唤起面板时的前台应用**（无切换跟踪）；粘贴前轮询确认应用真的回到前台（最多 2s），失败则 HUD「已复制，请手动粘贴」且**回滚"已粘贴"标记**（空回车恢复粘贴语义），绝不盲贴 |
| F40 | 原地替换方式 | 按上一次粘贴文本的字符数发退格删除，再 ⌘V（不用 ⌘Z——终端类应用不支持文本撤销）；退格分批发送防事件洪峰 |
| F41 | 剪贴板纪律 | 面板每轮结果留在剪贴板（设计如此）；划词/全选优化用完**恢复**用户原剪贴板（含图片等非文本） |
| F42 | 替换可还原 | 每次 F5/F8 替换都记录（替换前后文本+目标），供 F11 还原 |

## 已知边界（当前版本刻意不支持）

- 面板开着时点击别的应用/输入框，粘贴目标**不会**跟着走——目标永远是唤起时的前台应用。换目标 = 关面板重新唤起。（切换跟踪功能已于 2026-07-22 整体放弃，历史见 git `d15b047..fcbc878`）
- 每次唤起都是全新会话，无逐输入框会话记忆。
- 替换按"上次粘贴长度退格"，若用户在目标输入框手动改过已粘贴的内容，替换可能删错位置。
